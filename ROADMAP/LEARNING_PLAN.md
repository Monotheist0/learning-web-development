**Goal:** Break the "Copy-Paste" loop. Move to Next.js, but change **HOW** we learn.

### 1. The "Fade-Out" Protocol (AI Interaction)
We will change how you prompt the AI. You are forbidden from asking "Write code for X."
*   **Level 1 (Current):** "Write a search bar component." (‚õî STOP DOING THIS)
*   **Level 2 (The New Standard):** "I need a search bar. What variables do I need to track? Don't write the JSX, just list the steps."
*   **Level 3 (Goal):** "Here is my plan for the search bar. Does this data flow make sense?"

### 2. The "Box-Logic" Diagram (Before Code)
You cannot write code until you can "draw" it in plain English.
*   **Box:** What is the component?
*   **Memory:** What data lives here? (State)
*   **Wires:** Who does it talk to? (Props/Events)
*   *Drill:* Spend 5 minutes drawing the feature on paper/Excalidraw before opening VS Code.

### 3. The "Manual Override" Rule
*   **Absolutely NO Copy-Pasting Code.**
*   If the AI gives you a snippet, you must **read it, hide it, and type it from memory**.
*   If you forget syntax, peek for 2 seconds, hide it again, and type. This forces your brain to "hold" the syntax.

### 4. Next.js Strategy
*   We will move to Next.js to get a "fresh start."
*   We will ignore complex infrastructure (Postgres) for the first week and focus purely on **Layouts, Pages, and Data Flow** (using fake JSON data).
*   **Focus:** "How do I pass data from a Page to a Component?" (The Wires).
